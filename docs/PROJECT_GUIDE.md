# Project Guide — deep-research-agent

## 仓库定位

- 代号：dp
- 定位：DeepResearch 项目仓库

## 目标

- 完成检索-分析-归纳-输出最小闭环
- 输出结果带来源或证据说明
- 形成可复用研究流程模板

## 非目标

- 不做通用聊天产品壳层
- 不牺牲证据链换取短期速度

## 技术与实现侧重点

- Agent 编排、研究流程、证据链管理

## 第一阶段交付项

- 明确最小可运行闭环与输入输出
- 完成一轮可复现的验证记录
- 形成下一阶段的优化清单

## 完成标准

- 研究链路能稳定复现
- 关键结论可追溯到来源
- 流程改动在 design 或 changelog 有记录

## 当前目录约定

- docs/: 项目文档与规范
- docs/design/: 设计与重构方案
- docs/journal/: 过程记录
- src/: 代码实现

## 质量约束

- 变更可追踪：重要改动需在文档或变更记录中可回查
- 文档与实现一致：避免代码已变、文档未更新
- 提交前缀规范：feat / chore / fix / doc / refactor
